A picture of constructive mathematics

21 May 2021 Exotic expositions Type theory Constructive mathematics

There are three kinds of object in the world: algorithms, problems, and machines. An algorithm may solve a problem, a problem may be computed on a machine, and a machine may simulate another machine. If an algorithm \(x\) solves a problem \(X\), we write \(x\)\(:\,\)\(X\); if a problem \(X\) is computed on a machine \(\Gamma\), we write \(X\)\(:\,\)\(\Gamma\); if a machine \(\Gamma\) realizes a more general machine \(\wp _ {n}\), we write \(\Gamma\)\(:\,\)\(\wp _ {n}\). In each case, we say an object \(\xi\) is of type \(\Xi\).

We have countably infinite identifier symbols, which can denote any algorithms, problems and machines, without any predetermined meaning. We have countably infinite memory bits and countably infinite time slots. However, there are several fundamental rules we ought to obey.

In the very beginning, we possess a primitive machine \(\wp _ {0}\), which can only compute decidable problems. At any rate, we at least have \(\wp _ {0}\) at our disposal. Therefore a machine is valid, if it is stronger than \(\wp _ {0}\). Especially, put most plainly, if we have \(\wp _ {0}\), then we have \(\wp _ {0}\):

\(\;\mathtt {Primitive\_type}\;\)\(:\,\)\(\;\mathtt {has}\;\)\(\wp _ {0}\)

Furthermore, \(\wp _ {0}\) realizes a more general machine \(\wp _ {1}\), and \(\wp _ {1}\) realizes a more general machine \(\wp _ {2}\), and so on:

\(\wp _ {0}\)\(:\,\)\(\wp _ {1}\)\(;\,\)\(\wp _ {1}\)\(:\,\)\(\wp _ {2}\)\(;\,\)\(.\)\(.\)\(.\)\(;\,\)

We can define functions between objects. If algorithm \(x\) implements another algorithm \(Y\) under function \(\varphi\), we write \(\varphi\)\(\left[ x :\, X \right]\)\(\;=\;\)\(Y\). If problem \(X\) reduces another problem \(Y\) under function \(\varphi\), we write \(\varphi\)\(\left[ X :\, \wp _ {0} \right]\)\(\;=\;\)\(Y\). If machine \(\Gamma\) simulates another machine \(\Delta\) under function \(\varphi\), we write \(\varphi\)\(\left[ \Gamma \right]\)\(\;=\;\)\(\Delta\).

As a special case, once \(\Gamma\) doesn’t depend on additional objects than \(\Delta\) does, we get a valid new machine by combining \(\Gamma\)\(,\,\)\(\Delta\). Suppose computer \(\Gamma\) can compute problem \(X\) . With \(\Gamma\), and with any algorithm \(x\) which solves \(X\), we have a valid new computer stronger than \(\wp _ {0}\):

\(\;\mathtt {Function\_problem}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma \right]\)\(\;=\;\)\(X\)\(:\,\)\(\wp _ {0}\)\(,\,\)\(\;\mathtt {has}\;\)\(\left( \Gamma ,\, x :\, X \right)\)\(;\,\)

With \(\Gamma\), and with any problem \(X\) which is computed on \(\Delta\), we have a valid new computer stronger than \(\wp _ {0}\):

\(\;\mathtt {Function\_machine}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma \right]\)\(\;=\;\)\(\Delta\)\(,\,\)\(\;\mathtt {has}\;\)\(\psi\)\(\left[ \Gamma ,\, X :\, \Delta \right]\)\(:\,\)\(\wp _ {0}\)\(;\,\)

With a valid composite of machines \(\Gamma\)\(,\,\)\(\Delta\), once \(\Gamma\) doesn’t depend on additional objects than \(\Delta\) does, we can use only \(\Delta\):

\(\;\mathtt {Scope}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma ,\, \Delta \right]\)\(\;=\;\)\(\wp _ {0}\)\(,\,\)\(\;\mathtt {has}\;\)\(\psi\)\(\left[ \Gamma \right]\)\(:\,\)\(\Delta\)\(;\,\)

Rules FunctMachine and FunctProblem correspond to universal instantiation in logic.

Suppose we have such a function, that, with machine \(G\), and with any \(x\) an implementation of \(X\), we can simulate machine \(\Delta\) . Then, with \(G\), we may obtain a function, by which any \(x\) a solution of \(X\) gives rise to a way to simulate \(\Delta\) :

\(\;\mathtt {Elimination\_machine}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma ,\, x :\, X \right]\)\(\;=\;\)\(\Delta\)\(,\,\)\(\;\mathtt {has}\;\)\(\psi\)\(\left[ x :\, X \right]\)\(\;=\;\)\(\Delta\)\(;\,\)

Suppose we have such a function, that, with machine \(G\), and with any ^^.x^^ an implementation of \(X\), we can solve problem \(Y\) . Then, with \(G\), we may obtain a function, by which any \(x\) a solution of \(X\) gives rise to a way to solve \(Y\):

\(\;\mathtt {Elimination\_problem}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma ,\, x :\, X \right]\)\(\;=\;\)\(Y\)\(:\,\)\(\wp _ {0}\)\(,\,\)\(\;\mathtt {has}\;\)\(\psi\)\(\left[ x :\, X \right]\)\(\;=\;\)\(Y\)\(:\,\)\(\wp _ {0}\)\(;\,\)

Rules ElMachine and ElProblem correspond to the deduction theorem in logic.

Suppose on machine \(\Gamma\), any algorithm \(x\)\(:\,\)\(X\) gives rise to an object \(\xi\)\(:\,\)\(\Xi\) . Then on machine \(\Gamma\), this map can properly define a function; the function is written as \(\leftthreetimes _ {x :\, X}\)\(\xi\), of type written as \(\forall _ {x :\, X}\)\(\Xi\) :

\(\;\mathtt {Abstraction}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma ,\, x :\, X \right]\)\(\;=\;\)\(\xi\)\(:\,\)\(\Xi\)\(,\,\)\(\;\mathtt {has}\;\)\(\psi\)\(\left[ \Gamma \right]\)\(\;=\;\)\(\leftthreetimes _ {x :\, X}\)\(\xi\)\(:\,\)\(\forall _ {x :\, X}\)\(\Xi\)\(;\,\)

Suppose on machine \(\Gamma\), there is a function of type \(\forall _ {x :\, X}\)\(\Xi\) . Then on machine \(\Gamma\), this function can map any algorithm \(x\)\(:\,\)\(X\) to an object \(\xi\)\(:\,\)\(\Xi\), by replacing all occurrence of \(x\) in \(\Xi\) with \(y\) :

\(\;\mathtt {Application}\;\)\(:\,\)\(\;\mathtt {given}\;\)\(\varphi\)\(\left[ \Gamma \right]\)\(:\,\)\(\forall _ {x :\, X}\)\(\Xi\)\(,\,\)\(\;\mathtt {and}\;\)\(y\)\(:\,\)\(X\)\(,\,\)\(\;\mathtt {then}\;\)\(\varphi\)\(\left[ \Gamma ,\, y \right]\)\(:\,\)\(\Xi\)\(\left[ x \;\equiv\; y \right]\)\(;\,\)

Rules Abstraction and Application correspond to conditional proofs and modus ponens in logic.
