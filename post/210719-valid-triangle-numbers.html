Valid triangle numbers

19 July 2021 Exotic expositions Programming LeetCode Algorithm

Given a ordered set \(C\) of integers, with size \(N\), return the number of such triplets chosen from \(C\) that can make up triangles, if we take them as lengths of the side.

Constraints:

\(1\)\(\:\leq\:\)\(N\)\(\:\leq\:\)\(1\)\(0\)\(0\)\(0\)\(;\,\)

\(0\)\(\:\leq\:\)\(C _ {\left[ i \right]}\)\(\:\leq\:\)\(1\)\(0\)\(0\)\(0\)\(;\,\)

We want to find \(f\)\(\left[ C \right]\), the number of valid triples of triangle number. To slightly abuse notation, we may consider \(C\) to be an unordered set, when the order of elements doesn’t matter. The brute-force approach would be exhausting all triplets in \(C\), with complexity \(O\)\(\left[ N ^ {3} \right]\).

How can we do better? Let \(C\) be sorted, which takes \(O\)\(\left[ N \,\mathrm {log}\, N \right]\) . Without loss of generality, let the triplet be \(\left\langle x ,\, y ,\, z \right\rangle\) with \(x\)\(\:\leq\:\)\(y\)\(\:\leq\:\)\(z\). We only have to check:

\(x\)\(+\)\(y\)\(\:>\:\)\(z\)\(;\,\)

Split cases of whether the biggest component \(z\)\(\:\in\:\)\(C\) is chosen. Call \(D\)\(\:\equiv\:\)\(C\)\(\backslash\)\(\left\{ z \right\}\). If \(g\)\(\left[ z ,\, D \right]\) is the number of valid triples, in which \(z\) must be chosen, then:

\(f\)\(\left[ C \right]\)\(\;=\;\)\(g\)\(\left[ z ,\, D \right]\)\(+\)\(f\)\(\left[ D \right]\)

We see that \(g\)\(\left[ z ,\, D \right]\) has complexity \(O\)\(\left[ N \right]\) . Indeed, fix \(z\), for any \(x\), we have \(y\)\(\:>\:\)\(z\)\(-\)\(x\). Here \(r\)\(\left[ z - x \right]\), the number of such \(y\), may be found with a table, filled before the recursion. Hence, \(g\)\(\left[ z ,\, D \right]\) may be so defined:

\(g\)\(\left[ z ,\, D \right]\)\(\;=\;\)\(\displaystyle\sum\limits _ {x \:\in\: C}\)\(r\)\(\left[ z - x \right]\)\(;\,\)

The complexity of calling \(g\) is \(O\)\(\left[ N \right]\), and the complexity of exhausting \(x\) is also \(O\)\(\left[ N \right]\). The complexity of recursively calling \(g\) is \(O\)\(\left[ N ^ {2} \right]\), which overwhelms the complexity of sorting. The overall complexity is therefore \(O\)\(\left[ N ^ {2} \right]\).

class Solution:

 

  def count_triangle_number(self, choice: TYPE.List[int]) -> int:

    choice = [value for value in choice if value != 0]

    if not choice: return 0

    choice.sort()

    record = dict()

    for head in range(max(choice) * 2 + 1):

      record[head] = sum([1 for value in choice if value <= head])

    record[-1] = 0

    distinct = list(set(choice))

    distinct.sort()

    number = self.count_various(record, distinct)

    return number

 

  def count_various(self, record: TYPE.Dict[int, int], distinct: TYPE.List[int]) -> int:

    if not distinct: return 0

    small = distinct[0]

    partial = distinct[1:]

    number_fixed = self.count_fixed(record, small, partial)

    number_partial = self.count_various(record, partial)

    number = number_fixed + number_partial

    return int(number)

 

  def count_fixed(self, record: TYPE.Dict[int, int], small: int, distinct: TYPE.List[int]) -> int:

    number = 0

    novel_small = record[small] - record[small - 1]

    number += self.pick_three(novel_small)

    if not distinct: return number

    number += (record[small * 2 - 1] - record[small]) * self.pick_two(novel_small)

    for value in distinct:

      bound = value + small - 1

      novel_medium = record[value] - record[value - 1]

      if (novel_medium == 0): continue

      total_above = record[bound] - record[value]

      add_distinct = total_above * novel_medium

      add_pair_medium = self.pick_two(novel_medium)

      number += novel_small * (add_distinct + add_pair_medium)

    return int(number)

 

  def pick_two(self, number: int) -> int:

    if (number < 2): return 0

    return number * (number - 1) / 2

 

  def pick_three(self, number: int) -> int:

    if (number < 3): return 0

    return number * (number - 1) * (number - 2) / 6
